# Coach Engine | Cumulative Changelog (Stabilization Phase)

## Core System & Authentication
- **Hard-Logout Implementation**: Completely overhauled the logout sequence to ensure atomic termination of session states and cookies. Replaced fragmented, error-prone redirects with a forced `st.rerun()` pattern to prevent stale session persistence in production.
- **Cookie Controller Safeguard**: Implemented defensive `try-except` blocks and null-checks (`if controller:`) around all `CookieController` interactions. This eliminated the `TypeError` crashes that occurred when the browser component was not yet mounted during the initial script pass.
- **Log Event Consistency**: Standardized the event logging pipeline by auditing all `log_event` and `log_event_sync` calls. Verified imports across `app.py` to prevent `NameError` exceptions, ensuring consistent audit trailing for auth-related events.
- **Auto-Login Normalization**: Fixed the `KeyError` regarding `user_rol_id` vs `use_rol_id` to ensure dictionary keys perfectly match the database schema during user session object creation.

## Localization & UI
- **i18n Integrity Mapping**: Performed a full audit of `lang_eng.py` and `lang_swe.py`. Explicitly injected missing authentication error keys (`err_invalid_credentials`, `err_profile_missing`, `msg_login_success`) to resolve UI rendering of raw key strings.
- **Error Mapping Clarity**: Refactored the login logic in `login.py` (via `app.py` integration) to distinguish between:
    - **Credential Failures**: Auth provider rejection (wrong password).
    - **Data Integrity Failures**: Auth provider success but missing profile record in the local `users` table.

## Architectural Integrity
- **Circular Dependency Cleanup**: Audited `views/coach_architect.py` and resolved import loops that caused stability issues during module loading.
- **Session State Management**: Normalized `st.session_state` wiping during the logout flow. Used explicit `del` and `None` assignments to ensure no "ghost" authentication artifacts (like `authenticated=True`) survive a logout event.
- **Repository Stabilization**: Resolved the "sticky" production session issues by enforcing a strict ordering: `Clear Cookies` -> `Clear State` -> `Log Event` -> `Rerun`.
