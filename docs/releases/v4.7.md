# Release Notes: Sprint 4.7

## 1. ⚙️ EMOM Loop Size Parameter
* **Feature:** The `loop_size` configuration parameter (representing the total number of unique exercises in the rotation) is now completely modular and controllable directly from the user interface instead of being hardcoded to a fixed length of 4.
* **Functionality:** System administrators or coaches can now explicitly define how many minutes/stations the EMOM rolling block utilizes before cycling back to the initial template row (adjustable from 1–10 exercises, defaulting smoothly to 4). The session architecture engine and calculation loops inside `architect.py` scale dynamically to match this state.

## 2. 🍒 Cherry Pick Builder (Manual Protocol Selector)
* **Feature:** A completely separate manual configuration track has been seamlessly integrated into the Coach Architect layout alongside the existing fully automated generation protocols.
* **Functionality:** Empowers the user to manually search, select, and hard-lock specific exercises station-by-station (Tabata A/B pairings) or block-by-block (Triplets) directly from the interface.
* **Architecture:** Utilizes identical highly stable, 24-hour dygns-cached exercise bank lookups. The search list has been updated to dynamically inject localized equipment classifications in parentheses next to the movement title (e.g., *Kettlebell* or *Bodyweight*) instead of falling back to a generic fallback placeholder string.

## 3. 📊 Optimized Preview & Summary (Less is More)
* **Enhancement:** The preview panel nested under the "Architect Summary" banner has undergone a significant vertical consolidation to optimize screen real estate and mirror production-grade standards.
* **Layout Structure:** Bulky, vertical-heavy `st.metric` cards have been fully decoupled from the workspace. Instead, **Protocol Type** and **Estimated Duration** are cleanly presented inside a streamlined, single-row inline code specification grid (`code layout`), pulling the generated session block immediately into structural focus without requiring empty scrolling.

## 4. 🪲 Bug Fixes & Type-Safety Coercion
* **Fix:** Resolved a series of strict Pylance/Ruff validation alerts (`reportArgumentType`) by enforcing early explicit dictionary typing annotations for `chosen_map` and `exe_options` as `Dict[str, Optional[int]]` inside the rendering routines, preventing Python from inferring restrictive immutable types.
* **Fix:** Synchronized database field anomalies across core calculation modules to guarantee that primary equipment identifiers map flawlessly from localized lookups out into the interactive visual components.
