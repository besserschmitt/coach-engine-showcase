# Release Notes v2.0 – "The Core Rebirth"
**Release Date:** 2026-05-16

### 🚀 Overview
This major release marks the transformation of the application from a simple prototype into a stable, production-ready system. The primary focus was resolving critical connectivity barriers and engineering the foundational logic for physical session tracking.

### ✨ Key Features
* **Cookie-Based Persistence:** Integrated a dedicated `CookieController` utilizing the `ce_user_id` web cookie to solve authentication drops. Implemented a `time.sleep(0.4)` micro-pause to eliminate race conditions during async client hydration.
* **Pure Client-Side Tabata Engine:** Migrated the training timer module completely from server-side Python to an isolated client-side execution environment via `st.iframe` and native JavaScript. This delivers zero-polling execution entirely immune to field latency.
* **Conditional Lazy Loading:** Structured core navigation blocks using explicit `with tabs[X]:` conditional layouts, forcing underlying database operations to execute exclusively when a view is actively visible.
* **Audio Core Synchronization:** Introduced programmatic runtime sound synthesis featuring structural themes like "La Cucaracha", "Rocky", and "Europe".

### 🛠 Technical Debt & Patches
* Enforced hard boundaries on raw data aggregation streams using explicit `.limit(20)` pagination constraints to conserve cellular bandwidth.
* Standardized adaptive movement classification, shifting legacy designations to the definitive term **"förenklad övning"** for explicit tracking.
