# Release Notes v3.0 – "Internationalization & Scaling"
**Release Date:** 2026-05-20

### 🚀 Overview
Version 3.0 refactored the structural codebase to decouple user interface localization from core backend queries, optimizing network traffic and implementing rigid data boundaries.

### ✨ Key Features
* **i18n Localization Architecture:** Deployed a fully decoupled internationalization (i18n) pipeline separating language translation schemas from computational routines.
* **Persistent Selector Caching:** Wrapped core database lookups in `selector.py` inside a global caching layer (`@st.cache_data(ttl=3600)`) to drastically minimize relational database strain.
* **Cross-User State Bleed Isolation:** Mandated unique, dynamic key scopes (`key=f"widget_{ses_id}"`) on all multi-user toggles and input fields, eliminating the "sticky checkboxes" memory leak phenomenon.
* **Engine Refactoring:** Pruned legacy script objects (`logic_legacy.py` and `taunt.py`) to reduce file asset bloat.

### 🛠 Technical Debt & Patches
* Resolved an unhandled typing crash (`AttributeError: typing.Union`) inside `requirements.txt` to align execution constraints.
* Stabilized database connection resilience across high-latency routing protocols using an upgraded `https2` execution adapter.
