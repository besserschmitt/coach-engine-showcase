# Release Notes v4.0 – "Governance & Enterprise Audit"
**Release Date:** 2026-05-21

### 🚀 Overview
This release brought enterprise-grade telemetry, accountability trails, and programmatic governance layers to the environment, aligning the system with data control, compliance, and strict algorithmic session tracking.

### ✨ Key Features
* **Enterprise Audit Trail Integration:** Launched a native telemetry framework (`src/logger.py`) executing automated capture of authorization states, authentication exceptions, cookie operations, and raw database modifications inside `access_logs`.
* **GDPR Compliance Pipeline:** Introduced user-directed data portability mechanisms within `views/admin_profile.py`, allowing automated user history extraction and structure auditing.
* **Parametric Session Segmentation:** Formalized the workout programming logic under three rigid structural classifications: Monostructural (Level 1), Hybrid Structure (Level 2), and Multi-Segment/"The Gauntlet" (Level 3).
* **Environmental & Weather Intelligence:** Re-engineered spatial rule tracking by combining a localized weather data cache with geographic rules (e.g. automatically overriding ground-level layouts during extreme wind conditions).

### 🛠 Technical Debt & Patches
* **Hot-Reload Deadlock Mitigation:** Implemented explicit lazy importing within `src/logger.py` to prevent cyclic rendering deadlocks during hot-reloads.
* **Language Synchronization:** Completed the final harmonization split separating English parameters (`lang_eng.py`) from Swedish UI representations (`lang_swe.py`).
